Overview

Vibe Coding

Vibe coding uses AI tools and agents (in the IDE, console or web browser) to primarily generate code, review code and perform other code related tasks based on natural prompt by used. The term was coined by AI researcher Andrej Karpathy in early 2025. Vibe coding tools like Cursor or GitHub Copilot uses powerful LLM in the behind. While initially Vibe Coding tools were mostly in IDEs, now we see them in terminal / CLIs and web browsers also.

  • AI-powered code generation and completion
  • Intelligent code review and suggestions
  • Automated code refactoring
  • Real-time debugging assistance

Spec Driven Development

While traditional vibe coding directly generates code from user's natural prompt, spec driven development takes a more structured approach. It uses a 4 step process to generate code - Step 1: Generate requirement from user's natural prompt. Step 2: Generate system design from requirement. Step 3: Create a list of tasks to be implementated. Step 4: Lastly, generates code following the list of tasks.

  • Specification-based code generation
  • Automated test case creation
  • Requirements traceability
  • Compliance and standards enforcement
