Vibe Coding to Agentic DevOps: Reinventing Software Engineering

Adoping Vibe Coding and Agentic Software Engineering can help you in improving productivity and faster time to market. Vibe Coding tools can help solve complex challenges across the development lifecycle including building complex apps, creating design, reviewing code, explaining & debugging code.

Overview

Vibe coding uses AI tools and agents (in the IDE, console or web browser) to primarily generate code, review code and perform other code related tasks based on natural prompt by used. The term was coined by AI researcher Andrej Karpathy in early 2025

  • Vibe Coding tools like Cursor, Windsurf, AWS Kiro or GitHub Copilot uses powerful LLMs (like Claude, Gemini, GPT, Codex, Grok, local models) in the behind.
  • Vibe Coding tools could be IDE, terminal/CLI or browser based. They also could be agentic which means they do the entire work mostly independently.
  • Could be used to generate code, generate documentation, generate unit testing code, review code, create requirement and design doc.
  • Can generate code for backend, front end, UI, database and AI/ gen AI. Works for most programming languages, framework, libraries and databases.

Specifications (Requirements, Design, Tasks, ...)

While traditional vibe coding directly generates code from user's natural prompt, spec driven development takes a more structured approach. It uses a 4 step process to generate code - Step 1: Generate requirement from user's natural prompt. Step 2: Generate system design from requirement. Step 3: Create a list of tasks to be implementated. Step 4: Lastly, generates code following the list of tasks.

  • Specification-based code generation
  • Automated test case creation
  • Requirements traceability
  • Compliance and standards enforcement

Best practices

  • Be very explicit and make your ask detailed, while working with a Vibe Coding tool. Consider the Vibe Coding tool as a new employee and talk with it the way you would with a new employee.
  • Avoid providing step by step instructions, or giving technical steps. Avoid 'How' and focus on 'What'. This allows the tool be creative.
  • Give as much context as you can. This may include requirements, product description, product goals, design documents, UI wireframes, UX guidelines, any notes etc.
  • Few new Vibe Coding tools like AWS Kiro allows to create specification documents (like requirement and design) before starting coding. Also follow this approach for complex asks.
  • While the Vibe Coding tools have come a long way and they can perform complex tasks, it’s always better to make incremental changes so that it would be easier to understand, keep track and revert changes if required.
  • Use version control tools like git to preserve versions as sometime Vibe Coding tools produce incorrect ourput and it would be handy to revert to a previously saved version
  • For complex tasks (e.g. UI redesign or refactoring of existing code), ask the tool for few options, carefully judge the pros and cons of all the options, and then choose the optimum one to be implemented.
  • Always review the work done by the Vibe Coding agent. For example, after code changes are made do a review. Be aware of the changes made, code structure etc.
  • While it's good to be detailed oriented, avoid providing detailed steps, or giving technical steps. Avoid 'How' and focus on 'What'. This allows the tool be creative.
  • Vibe Coding tools have a tendency to generate complex code and large volume of code (which could be otherwise done in a simpler way), which can cause code sprawl. Tweak your prompts to ensure code reuse and composability.
  • Select the right Vibe Coding tool and LLM based on your requirement. While most of the time, having the tool select the best LLM works, sometimes you may want to explicitly choose the optimum LLM for the task
  • Be aware of the limitations of Vibe Coding tools. Always think about and keep an eye on architecture principles, security, maintainability of code generated etc.
  • Monitor the usage of the Vibe Coding tools to ensure that you are within the usage limits of your plan.
  • As the context grows, many a time Vibe Coding tools become slow. Creating a new session can help in such sitautions.
  • For enterprise adoption or large teams, having a guidance document with sample best prompts might come handy.
